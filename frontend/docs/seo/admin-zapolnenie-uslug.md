# Заполнение услуг из админки (тексты и SEO)

Город и регионы в примерах ниже: **офис в Сочи**, зона работ — **Краснодарский край, Ростовская область, Москва** (как в `.env`: `NEXT_PUBLIC_CITY`, `NEXT_PUBLIC_SERVICE_REGIONS`). Если у вас другие значения — замените в текстах вручную.

Бренд в примерах: **Гарант Монтаж** (`NEXT_PUBLIC_SITE_NAME`).

---

## Как устроена страница услуги в админке

**Путь:** `/admin/services` → открыть услугу по slug.

| Блок в форме | Что это на сайте |
|--------------|------------------|
| **Название** | Заголовок в шапке (hero), если в SEO не задан отдельный H1 |
| **Краткое описание** | Карточка на главной и списке `/services` |
| **Баннер лендинга** | Фон под заголовком услуги |
| **Тексты страницы услуги** | Блоки по порядку (как на публику) |

Блоки лендинга (сверху вниз):

1. **Микроразметка (schema)** — название и описание для поисковиков + диапазон цен (необязательно).
2. **Шапка страницы** — подзаголовок (лид), тег над заголовком, цели (goals), список фич.
3. **Витрины (showcase)** — подписи к секциям с фото (если фото не заданы, блоки могут скрываться).
4. **Текстовые колонки** — два абзаца рядом (иногда с акцентом).
5. **Боль / проблемы** — заголовок, пункты, вывод.
6. **Преимущества** — карточки с иконками.
7. **Шаги работы** — этапы.
8. **FAQ** — вопрос–ответ.

**Мета title / description / H1 для поиска** задаются отдельно: **`/admin/seo`** → страница с путём `/services/...` (не в карточке услуги).

## Самый быстрый способ заполнить длинный текст лендинга

1. Задеплойте на сервер версию сайта с актуальным кодом шаблонов (`service-landing-defaults.ts`).
2. В **`/admin/services`** откройте нужную услугу.
3. Нажмите **«Подставить шаблон»** (подтвердите замену) — подтянутся все блоки с текстами под семантику и гео.
4. **Сохраните**.
5. Заполните **SEO** по таблице ниже (если в БД ещё пусто или нужно перезаписать).

Если кнопку использовать нельзя — копируйте тексты из того же репозитория (функции `*Doc()` в `frontend/src/lib/service-landing-defaults.ts`) или из JSON внизу формы («JSON отладка») после подстановки шаблона локально.

## Про Москву (SEO текстами, без дорогого Директ)

В длинном тексте лендинга уже заложена формулировка выездных проектов по регионам. Для усиления Москвы в **отдельных абзацах или FAQ** можно добавить: «Работаем на объектах в Москве по согласованию графика и сметы» — **не** дублируя все города в одном title.

## Чеклист после сохранения

- [ ] Страница услуги открывается без ошибок, заголовок совпадает с задумкой (SEO H1 или «Название»).
- [ ] В `/admin/seo` для каждого `/services/...` заполнены title и description.
- [ ] Картинки баннера/обложки загружены при необходимости.
