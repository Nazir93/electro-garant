## Принципы

1. **Коммерческие страницы** (`/services/*`, `/services`) закрывают запросы с явным заказом («электромонтаж Сочи», «СКС Краснодар»). **Блог** берёт **информационный и смешанный интент**: «как выбрать», «что входит», «из чего цена», «ошибки при монтаже», сравнения протоколов — без дублирования главного SEO-текста услуги.
2. **Гео** в статьях — там, где это естественно (прайс по регионам, выезд, климат/фасады на Юге). Формулировка из памятки: офис в Сочи, проекты в Краснодарском крае, Ростовской области, Москве — без ложного «только Москва».
3. **Внутренние ссылки**: из каждой статьи — на 1–2 релевантные услуги и при возможности на кейс из портфолио.
4. **Частоты** в таблице ниже — ориентир из Вордстата (общие или региональные блоки); при публикации переснимите актуальные цифры.

## Шаблон мета для публикации в админке

- **Title (через PageMeta при необходимости):** `[Заголовок H1] | Гарант Монтаж` (или короче 55–60 знаков).
- **Description:** 1–2 предложения: польза + ключ без переспама + гео только если уместно.
- **Excerpt** карточки блога: 1–2 предложения, отличающиеся от description (можно вопрос пользователю).

---

## Структура текста (рекомендуемые H2)

Общий каркас для статей «как / что / почём»:

1. Вводная: проблема читателя + что получит после статьи.
2. Основной блок по пунктам (подзаголовки с **естественными** вхождениями LSI из Вордстата).
3. «Типичные ошибки» или «что спросить у подрядчика».
4. Блок с мягким CTA: «проектируем и монтируем под ключ» + ссылка на услугу.

HTML в БД: обычные `<p>`, `<h2>`, `<h3>`, `<ul>`, ссылки `<a href="/services/...">`.
